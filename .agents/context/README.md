# Context

`context/` holds repository facts agents need before making changes.

## Files

- `project-overview.md`: product direction and current implementation state.
- `runtime-and-deployment.md`: Nuxt, NuxtHub, storage, routes, environment, and build behavior.
- `database.md`: current schema and migration rules.

## Human Mirrors

- `../../README.md`
- `../../docs/README.md`
- `../../docs/config/README.md`
- `../../docs/server/README.md`
- `../../PLAN.md`

## Staleness Rule

If the codebase and these context files disagree, inspect the codebase and update the context files
in the same change.
